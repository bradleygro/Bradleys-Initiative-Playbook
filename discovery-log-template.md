## Purpose

This document tracks all discovery work for [Project Name]. It is a living artifact — updated as questions get answered, new questions surface, and decisions get made.

The goal of discovery is not to validate a pre-built solution. It is to answer four blocking questions well enough to define what to build before any production code is written:

1. **Value** — Will users actually use this?
2. **Usability** — Can they figure it out without help?
3. **Feasibility** — Can engineering build it?
4. **Viability** — Is it compliant, legally sound, affordable to support?
